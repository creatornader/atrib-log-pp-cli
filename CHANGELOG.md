# Changelog

All notable changes to `atrib-log-pp-cli` are documented in this file.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and the project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-05-17

### Added

- Initial release. Go CLI + paired MCP server, generated by [Printing Press](https://github.com/mvanhorn/cli-printing-press) from a hand-authored OpenAPI spec for `log.atrib.dev`.
- Command surface wrapping the public log API: `checkpoint`, `stats`, `recent`, `lookup`, `by-context`, `by-creator`, `tile`, `pubkey`, `entries` (submit), plus `sync` + `search` for the local SQLite mirror with FTS.
- Compound `workflow` and `analytics` commands for multi-step operations.
- Agent-friendly defaults: `--agent` flag, `--json` output, `--no-input` mode, explicit exit codes, `--dry-run` preview.
- MCP server (`atrib-log-pp-mcp`) wrapping the same surface for Claude Desktop / agent use.
- Apache 2.0 license.

### Positioning note

This CLI is the developer-facing surface for `log.atrib.dev`. It is *complementary* to the [`@atrib/*`](https://github.com/creatornader/atrib) MCP cognitive primitives (`emit`, `annotate`, `revise`, `recall`, `trace`, `summarize`), not a replacement. Use the MCP primitives for agent cognitive work; use this CLI for human or script interaction with the public log API.
